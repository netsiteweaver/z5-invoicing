1. Overview

The Z5 Invoicing System is a comprehensive business management platform that handles products, customers, orders, sales, payments, inventory, suppliers, departments, and user access. It provides dashboards, role-based permissions, approval workflows, and printable documents.

Key Features

  • Sales Management: Complete order-to-sale workflow with payment tracking
  • Inventory Control: Multi-department inventory with stock transfers and goods receipts
  • Payment Processing: Track payments and payment terms
  • Department Management: Organize business by departments/locations
  • Units of Measure: Flexible UOM system for different product types
  • Approval Workflows: Goods receipts and stock transfers require approval
  • Real-time Analytics: Dashboard with sales charts and low stock alerts
  • Notification System: Email notifications for key business events

2. Dashboard

The dashboard provides an overview of key business metrics and recent activities.

Dashboard Features

  • Statistics Cards: Total orders, sales, customers, products, revenue, and pending orders
  • Sales Chart: Visual representation of sales over time with filtering options (monthly, yearly, custom date range)
  • Low Stock Alert: Shows products that are running low on inventory
  • Recent Orders: Latest 5 orders with customer and product details
  • Recent Logins: Shows recent user login activities

Using the Dashboard

  • Click on any statistic card to navigate to the related section
  • Use the sales chart filters to view different time periods
  • Click on low stock items to go directly to inventory management
  • Recent orders can be clicked to view full order details

3. Access & Accounts

  • URL: Provided by your administrator

6. Suppliers

What you can do

7. Products (Catalog)

What you can do

  • Manage categories and brands

8. Orders

Create an order

  1. Orders → New
  2. Select a customer and add items. Order Status is set to draft by default, but can be changed to pending or confirmed.
  3. Save

Manage

  • Edit while allowed.
  • When the order is confirmed, it can be converted to a sale.

10. Sales

Create a sale

  • Direct: Sales → New
  • From Order: Orders → open order → Convert to Sale

Manage sales

  • View all sales with customer and payment status information
  • Edit sales before they are finalized
  • Track payment status (pending, partial, paid, overdue)
  • Record payments against sales

11. Payments

What you can do

  • Record payments against sales
  • View payment history and details
  • Track payment methods and reference numbers

Record a payment

  1. Go to Payments → Record Payment
  2. Select the sale to record payment against
  3. Enter payment date, amount, and payment type
  4. Add reference number and notes if needed
  5. Save the payment

Payment features

  • Payment Types: Cash, Bank Transfer, Check, Credit Card, etc.
  • Partial Payments: Record multiple payments for the same sale
  • Payment Status: Automatically updates sale payment status
  • Overpayment Protection: Prevents recording payments exceeding sale amount

12. Inventory

What you can do

  • View stock by product and department
  • Record stock movements (in/out/transfer)
  • Low stock view and stock reports

13. Goods Receipts

What you can do

  • Record received goods with multiple items, costs and VAT breakdown
  • Submit receipts for approval workflow
  • Approve receipts to update inventory
  • Print GRN (Goods Receipt Note) as PDF

Create a goods receipt

  1. Go to Inventory → Goods Receipts → Create
  2. Select department and supplier
  3. Enter receipt date and supplier reference
  4. Add items with quantities, unit costs, and UOM details
  5. Add batch numbers and expiry dates if applicable
  6. Save the receipt (status: submitted)

Approval workflow

  • Submitted: Initial state when goods receipt is created
  • Approved: Receipt is approved and inventory is updated
  • Only approved receipts update inventory levels
  • Approved receipts cannot be edited

Features

  • Multi-item Support: Add multiple products to one receipt
  • UOM Support: Handle different units of measure
  • VAT Calculation: Automatic tax calculation based on product settings
  • Batch Tracking: Record batch numbers and expiry dates
  • PDF Generation: Print professional GRN documents

14. Stock Transfers

What you can do

  • Create transfers between departments
  • Receive transfers at destination departments
  • Print transfer documents as PDF
  • Track transfer status and history

Create a stock transfer

  1. Go to Inventory → Stock Transfers → Create
  2. Select source and destination departments
  3. Enter transfer date and notes
  4. Add items with quantities and UOM details
  5. Save the transfer (status: in_transit)

Transfer workflow

  • In Transit: Initial state when transfer is created
  • Received: Transfer is received and inventory is updated
  • Stock is deducted from source only when transfer is received
  • Received transfers cannot be edited

Receive a transfer

  1. Go to the transfer detail page
  2. Click "Receive Transfer" button
  3. System automatically updates inventory at both departments
  4. Transfer status changes to "Received"

Features

  • Multi-item Support: Transfer multiple products in one transfer
  • UOM Support: Handle different units of measure
  • Automatic Inventory Update: Updates stock levels at both departments
  • PDF Generation: Print professional transfer documents
  • Status Tracking: Monitor transfer progress

15. Departments

What you can do

  • Create and manage departments/locations
  • Assign department managers
  • Set main department for default operations
  • Deactivate departments when no longer needed

Create a department

  1. Go to Departments → Create Department
  2. Enter department name and description
  3. Add address, phone, and email if needed
  4. Assign a manager from existing users
  5. Mark as main department if this is the primary location
  6. Save the department

Department features

  • Manager Assignment: Assign users as department managers
  • Main Department: Set one department as the default for operations
  • Status Management: Activate/deactivate departments
  • Inventory Tracking: Departments are used for inventory location tracking
  • Email Notifications: Department events trigger email notifications

Using departments

  • Departments are used for inventory location tracking
  • Goods receipts and stock transfers use departments
  • Each department maintains separate inventory levels
  • Main department is used as default for new operations

16. Payment Terms

What you can do

  • Create and manage payment terms for sales
  • Set default payment terms
  • Define different payment types (days, end of month, manual)
  • Deactivate payment terms when no longer needed

Create payment terms

  1. Go to Payment Terms → Create Payment Term
  2. Enter term name and description
  3. Select payment type (days, end of month, or manual)
  4. Set number of days if using "days" type
  5. Mark as default if this should be the standard term
  6. Save the payment term

Payment term types

  • Days: Payment due after specified number of days
  • End of Month (EOM): Payment due at end of the month
  • Manual: Payment terms handled manually

Features

  • Default Terms: Set one payment term as default for new sales
  • Flexible Configuration: Support for various payment scenarios
  • Status Management: Activate/deactivate payment terms
  • Sales Integration: Payment terms are used in sales to determine due dates

17. Units of Measure (UOM)

What you can do

  • Create and manage units of measure for products
  • Define conversion factors between different UOMs
  • Organize UOMs by dimension (count, weight, volume, length)
  • Deactivate UOMs when no longer needed

Create a UOM

  1. Go to Units of Measure → Create UOM
  2. Enter UOM name and code
  3. Select dimension (count, weight, volume, or length)
  4. Set conversion factor to base unit
  5. Define minimum increment if applicable
  6. Save the UOM

UOM dimensions

  • Count: Individual items, pieces, units
  • Weight: Kilograms, pounds, tons
  • Volume: Liters, gallons, cubic meters
  • Length: Meters, feet, yards

Features

  • Conversion Factors: Define how UOMs convert to base units
  • Multi-dimensional Support: Handle different types of measurements
  • Inventory Integration: UOMs are used throughout the inventory system
  • Flexible Configuration: Support for complex measurement scenarios

21. Documents & Printing

Printable Documents

  • Goods Receipt Notes (GRN): Print professional GRN documents with item details, costs, and VAT breakdown
  • Stock Transfer Documents: Print transfer documents for inter-department movements
  • Sales Invoices: Print sales documents (when invoice system is implemented)

Print Features

  • PDF Generation: All documents are generated as PDF files
  • Professional Layout: Clean, business-ready document formatting
  • Company Branding: Documents include company logo and information
  • Detailed Information: Complete item details, quantities, costs, and totals

How to Print

  1. Navigate to the document detail page (e.g., Goods Receipt or Stock Transfer)
  2. Click the "Print" button
  3. PDF will be generated and downloaded automatically
  4. Use your browser's print function or PDF viewer to print

Note: Ensure popups are allowed for the site to enable PDF downloads.

22. Tips & Troubleshooting

Common Issues

  • Can't log in: Use "Forgot Password" link; if still blocked, contact administrator
  • Missing menu options: Your role may not have permission for that feature
  • PDF downloads blocked: Allow popups for the site in your browser settings
  • Low stock alerts: Check the notification badge in the top navigation
  • Can't edit approved documents: Goods receipts and stock transfers become read-only after approval/receipt

Best Practices

  • Regular Backups: Ensure system administrator performs regular data backups
  • User Training: Train users on proper workflows, especially approval processes
  • Inventory Accuracy: Regularly reconcile inventory with physical stock counts
  • Payment Tracking: Record payments promptly to maintain accurate financial records
  • Department Organization: Set up departments properly before creating inventory records

Performance Tips

  • Use search and filters to quickly find records in large lists
  • Keep product catalogs organized with proper categories and brands
  • Set appropriate reorder points to avoid stockouts
  • Regularly review and clean up inactive records
